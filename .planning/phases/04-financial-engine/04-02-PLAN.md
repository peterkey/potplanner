       `advanceByFrequency(date: Date, frequency: BillFrequency): Date`:
       - `weekly`: `addDays(date, 7)`
       - `biweekly`: `addDays(date, 14)`
       - `four_weekly`: `addDays(date, 28)`
       - `monthly`: `addMonths(date, 1)`
       - `annual`: `addYears(date, 1)`

       `getBillOccurrences(bill: Bill, startDate: Date, endDate: Date): Date[]`:
       CRITICAL implementation detail -- use the STABLE BASELINE pattern from RESEARCH.md to prevent date drift:

       For `monthly` and `annual` frequencies:
       - Use period-count approach: iterate `i = 0, 1, 2, ...` and compute each occurrence as `addMonths(bill.nextDueDate, i)` or `addYears(bill.nextDueDate, i)`
       - NEVER chain `addMonths(previousOccurrence, 1)` -- this causes drift (Jan 31 -> Feb 28 -> Mar 28 instead of Mar 31)
       - Loop: start at i=0, compute candidate = addMonths(base, i), if candidate > endDate break, if candidate >= startDate push to results, increment i

       For `weekly`, `biweekly`, `four_weekly` frequencies:
       - Day-based addition is exact (no clamping), so chaining from previous occurrence is safe
       - Start at `bill.nextDueDate`, advance until >= startDate, then collect until > endDate

       `getMonthlyBillCost(bill: Bill, year: number, month: number): number`:
       - `const monthStart = new Date(year, month - 1, 1)`
       - `const monthEnd = endOfMonth(monthStart)` -- returns 23:59:59.999 (correct for <= comparison)
       - `const occurrences = getBillOccurrences(bill, monthStart, monthEnd)`
       - Return `occurrences.reduce((acc, _) => acc.plus(bill.amountPence), new Decimal(0)).toNumber()`
