**Step 2: Create `src/components/accounts/account-list.tsx`.**

`'use client'` component. Receives accounts as prop. Renders:

1. **Page header row:** `<h1>` "Accounts" (heading: `text-2xl font-semibold` = 24px) + "Add account" primary button (opens create dialog). Use `Plus` icon from `lucide-react` in the button.

2. **Table** using shadcn Table components with columns: Name | Balance | Actions (Edit, Delete).
   - Each row: account name in Name column, `£${(account.initialBalancePence / 100).toFixed(2)}` in Balance column, and icon buttons for Edit and Delete in Actions column.
   - Edit button: `<Button variant="ghost" size="icon">` with `<Pencil className="h-4 w-4" />` from `lucide-react`. `aria-label={`Edit ${account.name}`}`. Opens edit dialog by setting `editingAccount` state.
   - Delete button: `<Button variant="ghost" size="icon" className="text-destructive">` with `<Trash2 className="h-4 w-4" />`. `aria-label={`Delete ${account.name}`}`. Opens AlertDialog confirmation.

3. **Total footer row:** Use `<TableFooter>` containing a row with:
   - "Total" label in Name column (use `text-muted-foreground` for the label)
   - Sum formatted as `£${(accounts.reduce((sum, a) => sum + a.initialBalancePence, 0) / 100).toFixed(2)}` in Balance column using `text-xl font-semibold` (display style per UI-SPEC)
   - Empty Actions column

4. **Empty state:** When `accounts.length === 0`, show centered content instead of the table:
   - Heading: "No accounts yet" (`text-lg font-semibold`)
   - Body: "Add your first bank account to start tracking your balances." (`text-muted-foreground`)
   - Ghost variant "Add account" button that opens the same create dialog

5. **Delete confirmation:** Use `AlertDialog` component for each row.
   - Title: "Delete account?"
   - Description: `` This will permanently delete "${account.name}". This cannot be undone. ``
   - Cancel button text: "Keep account"
   - Confirm button text: "Delete account" with `variant="destructive"` -- changes to "Deleting..." while pending
   - On confirm: call `deleteAccountAction(account.id)` via an async onClick handler. Use `useState<boolean>` for pending state during delete.

State management:
- `useState<{ id: number; name: string; initialBalancePence: number } | null>(null)` for `editingAccount` -- when set, opens the edit dialog
- `useState<boolean>(false)` for `createDialogOpen` -- controls create dialog
- The AccountForm component (Task 2) is rendered inside `<Dialog>` components controlled by these state variables

Key implementation details:
- Import icons: `import { Pencil, Trash2, Plus } from 'lucide-react'`
- The create Dialog is controlled: `<Dialog open={createDialogOpen} onOpenChange={setCreateDialogOpen}>`
- The edit Dialog is controlled: `<Dialog open={editingAccount !== null} onOpenChange={(open) => !open && setEditingAccount(null)}>`
- AccountForm receives `onClose` callback that sets `createDialogOpen(false)` or `setEditingAccount(null)`
- Balance display converts pence to pounds: `(initialBalancePence / 100).toFixed(2)` -- conversion happens ONLY at display layer per CLAUDE.md

Create `src/components/accounts/account-form.tsx`.

`'use client'` component. Used inside a `Dialog` from account-list.tsx for both creating and editing accounts.

**Props:**
```typescript
interface AccountFormProps {
  account?: { id: number; name: string; initialBalancePence: number } | null
  onClose: () => void
}
```

When `account` is provided and non-null, it is edit mode. When `account` is null/undefined, it is create mode.

**Implementation:**

1. Use `useActionState` from `'react'` (NOT from `'react-dom'` -- removed in React 19):
   ```typescript
   import { useActionState, useEffect } from 'react'
   import { createAccountAction, updateAccountAction, type AccountActionState } from '@/app/actions/accounts'

   const action = account ? updateAccountAction : createAccountAction
   const [state, formAction, pending] = useActionState(action, {} as AccountActionState)
   ```

2. **Dialog structure:** Use `DialogHeader`, `DialogTitle`, `DialogFooter` from shadcn.
   - Title: "Edit account" when editing, "Add account" when creating (per UI-SPEC copywriting contract)

3. **Form fields wrapped in `<form action={formAction}>`:**
   - Hidden input for edit mode: `<input type="hidden" name="id" value={account?.id} />`
   - "Account name" label + `<Input name="name" required defaultValue={account?.name ?? ''} placeholder="e.g. Current Account" />`
   - "Balance (£)" label + `<Input name="balancePounds" type="number" step="0.01" min="0" defaultValue={account ? (account.initialBalancePence / 100).toFixed(2) : '0.00'} />`

4. **Error display:** If `state.error`, render `<p className="text-sm text-destructive">{state.error}</p>` below the form fields.

5. **Submit button in DialogFooter:**
   - Create mode label: "Add account" / "Saving..." when pending
   - Edit mode label: "Save changes" / "Saving..." when pending
   - `disabled={pending}` attribute
   - `variant="default"` (primary violet)

6. **Dialog close on success via useEffect:**
   ```typescript
   useEffect(() => {
     if (state.success) {
       onClose()
     }
   }, [state.success, onClose])
   ```

Key rules:
- `useActionState` imported from `'react'` -- NOT `'react-dom'`
- The Server Action signature `(prevState, formData)` means `useActionState` passes prevState automatically -- the form just submits FormData
- Balance field uses `name="balancePounds"` matching what the Server Action reads via `formData.get('balancePounds')`
- Balance display converts pence to pounds for defaultValue: `(initialBalancePence / 100).toFixed(2)`
- `step="0.01"` allows penny precision in the input
- `min="0"` prevents negative balance entry
- DialogFooter wraps the submit button for proper positioning
