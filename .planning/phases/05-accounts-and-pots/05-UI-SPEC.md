## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 |
| Label | 14px (0.875rem) | 500 (medium) | 1.4 |
| Heading | 24px (1.5rem) | 600 (semibold) | 1.2 |
| Display | 28px (1.75rem) | 700 (bold) | 1.15 |

Notes:
- Body is the default prose and table cell size. Geist renders cleanly at 14px.
- Label applies to form labels, table column headers, and summary rows ("Total:", "Allocated:").
- Heading applies to page-level `<h1>` elements ("Accounts", "Pots") and dialog titles.
- Display applies to the total balance figure on the accounts page and the allocation counter on the pots page (e.g. "£2,450.00").
- Monetary amounts in table cells use body size (14px) at regular weight. Total rows use label weight (500).
- All font sizes use `rem` units to respect user browser preferences.

Source: Geist font confirmed in `globals.css` (`--font-sans`). Sizes are defaults.

| Role | Value | Usage |

| Dominant (60%) | `--background`: oklch(1 0 0) — white | Page background, dialog background |

| Destructive | `--destructive`: oklch(0.577 0.245 27.325) — red | Delete account button, delete pot button, destructive confirm dialog |

Accent (`--primary` violet) reserved for:
- "Add Account" primary CTA button
- "Add Pot" primary CTA button
- "Save" / "Update" submit button inside dialogs
- Active nav link indicator for `/accounts` and `/pots`
- Focus ring on form inputs and interactive controls

Accent is NOT used on: table row hover states, secondary action buttons (Edit), informational badges.

Secondary semantic color:
- `--destructive` (red): delete buttons and confirmation dialogs only. No other usage.

Warning state (over-allocation banner on pots page):
- Use shadcn `Alert` with `variant="default"` and amber/yellow tint via Tailwind utility `border-yellow-400 bg-yellow-50 text-yellow-900`. This is a non-blocking warning, not an error.

Source: CSS variables confirmed in `src/app/globals.css`.

## Interaction Contracts

### Accounts Page

**Account list table columns:** Name | Balance | Actions (Edit, Delete)

**Add account flow:**
1. User clicks "Add account" button
2. shadcn Dialog opens with create form (name + balance fields)
3. Submit triggers Server Action via `useActionState`
4. On success: dialog closes (via `useEffect` watching `state.success`), table row appears (via `revalidatePath`)
5. On error: error message displays inline beneath the offending field or as a form-level alert

**Edit account flow:**
1. User clicks Edit icon button on a table row
2. Same Dialog opens pre-populated with existing values
3. Submit triggers update Server Action
4. On success: dialog closes, row updates in place
5. On error: error message displays inline

**Delete account flow:**
1. User clicks Delete icon button on a table row
2. shadcn AlertDialog opens with confirmation copy
3. User clicks "Delete account" confirm button
4. Delete Server Action fires
5. On success: dialog closes, row removed from table
6. On error: error toast or inline message

**Total row:** Non-interactive. Renders below the last account row. Label "Total" in label weight; sum in display weight. Updates reactively via `revalidatePath`.

**Empty state:** Displayed when account list is empty. Centered in the table area. Shows heading, body copy, and a ghost variant of the "Add account" button.

### Pots Page

**Income input:** Plain number input at top of page. Stored in client `useState` only — no DB persistence in Phase 5. Labeled "Monthly income (£)". Defaults to 0. Visible note below: "Income resets when you reload the page."

**Allocation summary line:** Rendered directly below the income input. Updates reactively as user edits allocation inputs. Format: "Allocated: £X.XX / Income: £Y.YY". Color: default text when within budget; amber (`text-yellow-700`) when over-allocated.

**Over-allocation warning:** shadcn `Alert` component. Shown only when `validatePotAllocations` returns `valid: false`. Variant: warning styling (amber). Not a blocker — user can still save.

**Pot list table columns:** Name | Allocation (£) | Balance | Actions (Edit, Delete)

**Inline allocation editing:** Allocation column renders a number input directly in the table cell. On blur or form submit, triggers update Server Action. Each row's balance updates after save.

**Add pot flow:** Same Dialog pattern as accounts.

**Edit pot flow:** Dialog with name, allocation, and rollover toggle. Rollover toggle renders as shadcn Switch. Helper text beneath toggle explains it is a v2 feature.

**Delete pot flow:** Same AlertDialog confirmation pattern as accounts.

**Reset month flow:**
1. User clicks "Reset month" button (secondary variant, positioned at page header right)
2. AlertDialog opens with reset confirmation copy
3. On confirm: `resetPotAllocations()` Server Action fires
4. On success: all allocation inputs reset to 0, balance display resets, dialog closes
5. On error: error message in dialog

**Empty state:** Same pattern as accounts.

### Loading / Pending States

- Submit buttons: disabled, label changes to "Saving..." or "Resetting..." during pending (`useFormStatus` or `useActionState` pending flag)
- Delete buttons: disabled during pending, label changes to "Deleting..."
- No skeleton loaders in Phase 5 — Server Components render synchronously with data; no streaming needed

## Layout

### Nav (added to `(app)/layout.tsx`)

Minimal horizontal top nav bar:
- Left: app name "PotPlanner" (bold, primary color)
- Center/Right: nav links — Accounts | Pots
- Active link: underline or left border indicator in primary violet
- Height: 56px (14 × 4px units)
- Background: `--sidebar` (near-white, `oklch(0.985 0 0)`)
- Bottom border: `--border` (light grey)

### Page Layout

Both `/accounts` and `/pots` pages use the same layout shell:

```
[top nav — 56px]
[page content area]
  [page header row — h1 title + primary CTA button — 32px vertical padding]
  [summary row — totals/allocation counter — 16px vertical padding]
  [table or empty state]
```

Page content horizontal padding: 24px (lg) on mobile, 32px (xl) on desktop (md breakpoint and above).
Max content width: `max-w-4xl` (896px) centered.
