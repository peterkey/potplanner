**Step 2: Create `src/components/pots/pot-list.tsx`.**

`'use client'` component. This is the most complex component in Phase 5. It renders:

1. **Page header row:** `<h1>` "Pots" (heading: `text-2xl font-semibold`) on the left. On the right, two buttons:
   - "Reset month" secondary button (`variant="outline"`) -- opens reset confirmation AlertDialog
   - "Add pot" primary button (`variant="default"`) with `Plus` icon -- opens create dialog

2. **Income input section** (above allocation summary):
   - Label: "Monthly income (£)" (`text-sm text-muted-foreground`)
   - `<Input type="number" step="0.01" min="0" defaultValue="0.00" />` -- controlled via `useState<number>(0)` storing pence value
   - On change: parse input pounds to pence via `Math.round(parseFloat(e.target.value) * 100)`, update state
   - Below input, small muted note: "Income resets when you reload the page. Persistent income settings arrive in a future update."

3. **Allocation summary line** (below income input):
   - Format: `Allocated: £${(totalAllocatedPence / 100).toFixed(2)} / Income: £${(incomePence / 100).toFixed(2)}`
   - Use `text-xl font-semibold` for the display style (per UI-SPEC display typography)
   - Color: default text color when `validatePotAllocations(incomePence, potsForEngine).valid === true`; `text-yellow-700` when over-allocated
   - Compute `totalAllocatedPence` using `sumPotAllocations()` from engine -- pass the pots array mapped to the `Pot` interface shape

4. **Over-allocation warning** (only when `validatePotAllocations` returns `valid: false`):
   - Use shadcn `Alert` component with custom amber styling: `className="border-yellow-400 bg-yellow-50 text-yellow-900"`
   - AlertTitle: "Over-allocated"
   - AlertDescription: `Total allocation (£${(totalAllocatedPence / 100).toFixed(2)}) exceeds monthly income (£${(incomePence / 100).toFixed(2)}). You can still save, but review your pots.`

5. **Pot table** using shadcn Table components with columns: Name | Allocation (£) | Balance | Actions (Edit, Delete)
   - Each row:
     - Name column: pot name
     - Allocation column: `£${(pot.allocatedPence / 100).toFixed(2)}`
     - Balance column: `Balance: £${(getPotBalance(pot.allocatedPence, 0) / 100).toFixed(2)}` -- use `getPotBalance` from engine with `spentPence=0` for Phase 5
     - Actions: Edit button (`Pencil` icon, `variant="ghost" size="icon"`, `aria-label="Edit {pot name}"`) and Delete button (`Trash2` icon, `variant="ghost" size="icon" className="text-destructive"`, `aria-label="Delete {pot name}"`)

6. **Delete confirmation:** AlertDialog pattern identical to accounts:
   - Title: "Delete pot?"
   - Description: `This will permanently delete "${pot.name}". This cannot be undone.`
   - Cancel: "Keep pot"
   - Confirm: "Delete pot" with `variant="destructive"`, "Deleting..." while pending
   - On confirm: call `deletePotAction(pot.id)`

7. **Reset month confirmation:** AlertDialog triggered by "Reset month" button:
   - Title: "Reset all pot allocations?"
   - Description: "This will set all pot allocations to £0.00. Your pot names and settings will be kept."
   - Cancel: "Keep allocations"
   - Confirm: "Reset allocations" with `variant="destructive"`, "Resetting..." while pending
   - On confirm: call `resetPotAllocationsAction()`

8. **Empty state:** When `pots.length === 0`, show centered content instead of table:
   - Heading: "No pots yet" (`text-lg font-semibold`)
   - Body: "Create your first budget pot to start allocating your income." (`text-muted-foreground`)

Create `src/components/pots/pot-form.tsx`.

`'use client'` component. Used inside a `Dialog` from pot-list.tsx for both creating and editing pots.

**Props:**
```typescript
interface PotFormProps {
  pot?: { id: number; name: string; allocatedPence: number; rollover: boolean } | null
  onClose: () => void
}
```

When `pot` is provided and non-null, it is edit mode. When `pot` is null/undefined, it is create mode.

**Implementation:**

1. Use `useActionState` from `'react'` (NOT from `'react-dom'`):
   ```typescript
   import { useActionState, useEffect } from 'react'
   import { createPotAction, updatePotAction, type PotActionState } from '@/app/actions/pots'

   const action = pot ? updatePotAction : createPotAction
   const [state, formAction, pending] = useActionState(action, {} as PotActionState)
   ```

2. **Dialog structure:** Use `DialogHeader`, `DialogTitle`, `DialogFooter` from shadcn.
   - Title: "Edit pot" when editing, "Add pot" when creating (per UI-SPEC copywriting contract)

3. **Form fields wrapped in `<form action={formAction}>`:**
   - Hidden input for edit mode: `<input type="hidden" name="id" value={pot?.id} />`
   - "Pot name" label + `<Input name="name" required defaultValue={pot?.name ?? ''} placeholder="e.g. Groceries" />`
   - "Monthly allocation (£)" label + `<Input name="allocationPounds" type="number" step="0.01" min="0" defaultValue={pot ? (pot.allocatedPence / 100).toFixed(2) : '0.00'} />`
   - Rollover toggle section:
     - Container div with flex layout: label on left, Switch on right
     - Label text: "Roll over unspent amount"
     - Helper text below: "When enabled, unspent balance carries forward each month (v2 feature -- saved but inactive)" in `text-xs text-muted-foreground`
     - Use shadcn `<Switch name="rollover" defaultChecked={pot?.rollover ?? false} />`
     - The Switch component submits "on" as the form value when checked, which the Server Action parses as `formData.get('rollover') === 'on'`

4. **Error display:** If `state.error`, render `<p className="text-sm text-destructive">{state.error}</p>` below the form fields.

5. **Submit button in DialogFooter:**
   - Create mode label: "Add pot" / "Saving..." when pending
   - Edit mode label: "Save changes" / "Saving..." when pending
   - `disabled={pending}` attribute
   - `variant="default"` (primary violet)

6. **Dialog close on success via useEffect:**
   ```typescript
   useEffect(() => {
     if (state.success) {
       onClose()
     }
   }, [state.success, onClose])
   ```

Key rules:
- `useActionState` imported from `'react'` -- NOT `'react-dom'`
- Allocation field uses `name="allocationPounds"` matching what the Server Action reads via `formData.get('allocationPounds')`
- Allocation display converts pence to pounds for defaultValue: `(allocatedPence / 100).toFixed(2)`
- `step="0.01"` allows penny precision in the input
- `min="0"` prevents negative allocation entry
- Switch `name="rollover"` -- shadcn Switch is built on Radix and renders a hidden input that submits "on" when checked
- The rollover toggle is visible and functional (persists to DB) but the rollover logic itself is inert in Phase 5 -- the helper text makes this clear to the user
