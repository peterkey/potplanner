    ## Section 3: Database Rules
    - **INTEGER PENCE MANDATE**: All monetary values stored as `integer` (pence). NEVER use `decimal`, `numeric`, `real`, or `float` for money. Convert to pounds/currency only at the UI display layer. Column naming convention: `*_pence` suffix (e.g., `amount_pence`, `balance_pence`, `target_pence`).
    - **APPEND-ONLY LEDGER**: The `transfer_history` table is append-only. Rows are NEVER updated or deleted. Balances are derived by summing ledger entries, never stored as a mutable column. No table may have a `current_balance` column that gets updated in place.
    - **Interest rates**: Stored as integer basis points (2500 = 25.00%). Divide by 10000 for decimal representation.
    - **Migrations**: Run `npm run db:generate` to create SQL migration files, `npm run db:migrate` to apply them.

    ## Section 4: Security Rules
    - **`verifySession()` in every DAL function**: Every function in `src/lib/dal/` must call `verifySession()` before touching the database. This is the REAL security gate — not proxy.ts.
    - **`proxy.ts` is redirect-only**: It checks for a session cookie and redirects unauthenticated users to `/login`. It does NOT validate tokens or enforce authorization. Security enforcement happens in the DAL via `verifySession()`.
    - **JWT in httpOnly cookie ONLY**: Never expose JWT to JavaScript. Never send JWT in response body. Always set `httpOnly: true`, `secure: true` (in production), `sameSite: 'lax'`.
    - **`server-only` import**: Every file in `src/lib/db/`, `src/lib/dal/`, `src/lib/auth/`, and `src/server/` must have `import 'server-only'` as its first import. This causes a build-time error if the file is accidentally bundled for the client.

    ## Section 5: Coding Conventions
    - TypeScript strict mode — no `any`, no `// @ts-ignore` without explanation
    - Import alias: `@/` maps to `src/`
    - File naming: `kebab-case` for files, `PascalCase` for React components
    - Async request APIs: Always `await cookies()`, `await headers()`, `await params`, `await searchParams` (Next.js 16 requirement)
    - No custom webpack config — Turbopack is the default bundler in Next.js 16. If custom webpack is needed, pass `--webpack` flag.

    ## Section 6: Testing
    - **Vitest** for unit tests — config in `vitest.config.mts`
    - **Playwright** for E2E tests (configured in Phase 2)
    - **What to test**: Engine functions (pure logic, high coverage), API route handlers (request/response contracts), DAL functions (with test database). Do NOT unit test React component rendering unless business logic is embedded.
    - **Run tests**: `npx vitest run` (unit), `npx playwright test` (E2E)
    - **Type check**: `npm run type-check` (runs `tsc --noEmit`)

    ## Section 7: Development Workflow
    - **Local dev (without Docker)**: `npm run dev` — requires local PostgreSQL and Redis
    - **Local dev (with Docker)**: `docker compose -f docker-compose.yml -f docker-compose.dev.yml up app db redis`
    - **Production build**: `docker compose up --build`
    - **Generate migration**: `npm run db:generate` after editing `src/lib/db/schema.ts`
    - **Apply migration**: `npm run db:migrate`
    - **Type check before commit**: `npm run type-check`
    - **GSD workflow**: Use `/gsd:plan-phase` for planning, `/gsd:execute-phase` for execution, `/gsd:verify-work` for verification
    - **MCP tools**: Context7 for live library docs, GitHub MCP for repo operations, Playwright MCP for browser testing

    > **CRITICAL RULES** (breaking these causes cascading failures):

    > 4. import 'server-only' in ALL db/dal/auth/server files
    > 5. Use proxy.ts NOT middleware.ts (Next.js 16)
