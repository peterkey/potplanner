  <action>
    Create `docker-compose.dev.yml` â€” a development override file used with:
    `docker compose -f docker-compose.yml -f docker-compose.dev.yml up`

    ```yaml
    services:
      app:
        build:
          context: .
          target: dependencies
        command: npm run dev
        ports:
          - "3000:3000"
        volumes:
          - .:/app
          - /app/node_modules
          - /app/.next
        environment:
          NODE_ENV: development
    ```

    Key design decisions:
    - `target: dependencies` â€” stops the multi-stage build at the first stage (no production build needed for dev)
    - `command: npm run dev` â€” overrides the CMD from Dockerfile to run Next.js dev server
    - Volume mount `.:/app` â€” enables hot-reload by syncing host source code into container
    - Anonymous volumes `/app/node_modules` and `/app/.next` â€” prevents host OS node_modules and .next from shadowing container versions (avoids cross-platform binary conflicts)
    - `ports: "3000:3000"` â€” exposes dev server directly (no Nginx in dev)
    - Nginx service is NOT included in this file, so it remains defined only in docker-compose.yml (prod). When using the override file, explicitly do NOT start nginx: the override file simply doesn't redefine it, but docker-compose merges both files so nginx will still appear. To truly skip nginx in dev, users should run:
      `docker compose -f docker-compose.yml -f docker-compose.dev.yml up app db redis`
      Document this in CLAUDE.md (plan 04).
  </action>
