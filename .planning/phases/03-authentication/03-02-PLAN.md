**Create `src/app/api/auth/login/route.ts`:**
Export an async `POST(request: Request)` function:

1. Parse body: `const { email, password } = await request.json()`
2. Validate: if !email or !password, return `NextResponse.json({ error: 'Email and password are required' }, { status: 400 })`
3. Rate limit check (BEFORE any DB work):
   ```typescript
   const headerStore = await headers()
   const ip = headerStore.get('x-forwarded-for') ?? headerStore.get('x-real-ip') ?? 'unknown'
   try {
     await loginRateLimiter.consume(ip)
   } catch (rateLimiterRes) {
     if (rateLimiterRes instanceof RateLimiterRes) {
       return NextResponse.json(
         { error: 'Too many login attempts. Try again later.' },
         { status: 429, headers: { 'Retry-After': String(Math.ceil(rateLimiterRes.msBeforeNext / 1000)) } }
       )
     }
     throw rateLimiterRes
   }
   ```
4. Look up user: `const user = await getUserByEmail(email)`
5. If no user, return `NextResponse.json({ error: 'Invalid credentials' }, { status: 401 })`
6. Compare password: `const valid = await bcrypt.compare(password, user.passwordHash)`
7. If invalid, return 401 with same error message (do NOT reveal whether email exists)
8. Sign session: `await signSession(user.id)`
9. Return `NextResponse.json({ success: true })`

Import `bcrypt` from `bcryptjs`, `{ NextResponse }` from `next/server`, `{ headers }` from `next/headers`, `{ RateLimiterRes }` from `rate-limiter-flexible`.

Do NOT add `import 'server-only'` to route handlers — they are inherently server-only in the App Router.

Do NOT import `server-only` in seed script — it runs standalone via tsx, not in Next.js.
